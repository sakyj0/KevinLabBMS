## 테스트 완료 후 Git 커밋 가이드

각 기능 개발 완료 후 테스트를 진행하고 Git에 커밋하세요. 아래 커밋 메시지 컨벤션을 따라주세요.

```
<type>: <subject>

<body>

<footer>
```

### 커밋 타입
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 변경
- **style**: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- **refactor**: 코드 리팩토링
- **test**: 테스트 코드 추가 또는 수정
- **chore**: 빌드 프로세스, 패키지 매니저 설정 등 (소스 변경 없음)

### 예시
```
feat: 로그인 페이지 UI 구현

- 로그인 폼 레이아웃 구현
- 이메일, 비밀번호 입력 필드 구현
- 유효성 검사 로직 추가
- 반응형 디자인 적용

Closes #123
```
